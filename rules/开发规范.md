## 开发环境规范

- 禁止使用 PowerShell 5，必须使用 PowerShell 7；执行 PowerShell 命令时必须显式使用 `pwsh`。

## 代码质量规范

- Rust 代码的中文注释占比不得少于 1/4。

## 设计和开发流程规范

- 优先使用 Type Driven Development 进行设计；如果会导致设计过于复杂，则不要强行使用。

- 开发过程遵循 Type Driven Design + Test Driven Development：先定义必要的 `struct`、`enum`、函数签名和必要的 `trait`，再编写单元测试，最后完成具体实现；如果测试暴露出类型设计不合理，应回到类型定义阶段调整。

## 开发收尾规范

- 代码实现完毕且单元测试都通过后，要检查 `/requirements` 下对应的需求文档和 `/docs` 下对应的方案是否需要更新。
