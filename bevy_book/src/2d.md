# 一、修订说明

本文严格按照 **Bevy 0.19** 的组件式 API 编写。Bevy 0.19 的官方示例通常直接组合组件，而不再采用旧教程中常见的各种 Bundle 写法。例如：

```rust
commands.spawn(Camera2d);

commands.spawn((
    Sprite::from_color(Color::WHITE, Vec2::splat(64.0)),
    Transform::default(),
));
```

因此，旧稿中以下写法已全部移除：

- `Camera2dBundle`
- `SpriteBundle`
- `NodeBundle`
- `ButtonBundle`
- `TextBundle`
- `Camera3dBundle`
- `PbrBundle`
- `MaterialMesh2dBundle`
- `Mesh2dHandle`

Bevy 0.19 官方 Sprite 示例直接生成 `Camera2d` 和 `Sprite`；UI 示例直接使用 `Node`、`Button`、`Text`；2D Mesh 使用 `Mesh2d` 与 `MeshMaterial2d`；3D 使用 `Camera3d`、`Mesh3d` 与 `MeshMaterial3d`。

此外，还修正了以下错误：

1. 图集布局应使用 `TextureAtlasLayout::from_grid` 创建，而不是 `TextureAtlas::from_grid`。
2. `Anchor` 没有被移除，它现在是独立组件，不是 `Sprite` 的字段。
3. UI 不是世界坐标系统，而是基于 `Node`、百分比、像素、Flexbox 和 Grid 的布局系统。
4. UI 绘制顺序不能简单等同于 Sprite 的 `Transform.translation.z`。
5. 2D 不一定“没有深度测试”，更准确的说法是：Sprite 通常主要通过 Z 坐标和透明排序控制视觉层级。

---

# 二、Bevy 2D 系统是什么

Bevy 的 2D 系统不是一个独立于 ECS 的“2D 引擎”，而是一组用于构建二维场景的组件、资源和渲染管线，主要包括：

- `Camera2d`
- `Sprite`
- `Transform`
- `Text2d`
- `Mesh2d`
- `MeshMaterial2d<ColorMaterial>`
- `TextureAtlas`
- `TextureAtlasLayout`
- `TilemapChunk`
- 2D Picking
- 外部 2D 物理引擎

一个最小的 2D 场景通常只需要：

```rust
commands.spawn(Camera2d);

commands.spawn(Sprite::from_color(
    Color::srgb(0.2, 0.7, 1.0),
    Vec2::new(100.0, 100.0),
));
```

官方最小示例也是通过 `Camera2d` 和 `Sprite::from_image` 创建场景。

# 三、2D 系统的典型使用场景

## 3.1 纯 2D 游戏

适合：

- 平台跳跃游戏
- 俯视角射击游戏
- 棋牌和休闲游戏
- 塔防游戏
- RPG 地图
- 横版动作游戏
- 2D 沙盒游戏

典型实体组成：

```text
玩家实体
├── Sprite
├── Transform
├── Player
├── Velocity
├── Health
└── Collider
```

其中：

- `Sprite` 负责外观。
- `Transform` 负责位置、旋转、缩放和 Z 层级。
- `Player` 是自定义标记组件。
- `Velocity`、`Health` 是游戏逻辑组件。
- `Collider` 通常来自 Avian 或 Rapier。

---

## 3.2 2D 编辑器和可视化工具

适合：

- 节点编辑器
- 地图编辑器
- 图形标注工具
- 电路和流程图
- 科学数据可视化
- 关卡编辑器

这类程序通常会使用：

- `Mesh2d` 绘制几何形状
- `Text2d` 绘制跟随世界的标签
- `Camera::viewport_to_world_2d` 将鼠标窗口坐标转换成世界坐标
- UI 系统实现工具栏和属性面板

官方示例通过 `viewport_to_world_2d` 将鼠标位置转换成世界坐标，并展示了相机移动、缩放和自定义 Viewport。

---

## 3.3 瓦片地图

适合：

- RPG 地图
- Roguelike 地牢
- 沙盒地形
- 策略游戏地图
- 大型像素世界

Bevy 0.19 提供内置的：

```rust
TilemapChunk
TilemapChunkTileData
TileData
```

一个 Chunk 可以用单次绘制调用渲染整块瓦片，比“一个瓦片一个 Sprite 实体”的方式更适合大型地图。

对于动态对象，例如玩家、敌人和掉落物，仍然建议单独使用普通 ECS 实体。

---

## 3.4 世界空间文字

适合：

- 角色名称
- 伤害数字
- 地图标签
- 对话气泡
- 调试信息
- 血量文字

应使用：

```rust
Text2d
```

而不是 UI 的：

```rust
Text
```

`Text2d` 位于游戏世界中，会随相机移动、缩放和旋转；UI `Text` 通常位于屏幕布局中，不随世界相机移动。Bevy 官方文档明确区分了世界中的 `Text2d` 与独立于世界视口的 UI Text。

---

## 3.5 2D 特效与程序化图形

适合：

- 激光
- 轨迹
- 光圈
- 选择框
- 雷达范围
- 简单粒子
- 程序化几何图形

简单矩形和贴图优先使用 `Sprite`。

需要自定义几何体或材质时使用：

```rust
Mesh2d
MeshMaterial2d<ColorMaterial>
```

Bevy 0.19 官方示例使用如下组件绘制 2D 网格：

```rust
(
    Mesh2d(meshes.add(Rectangle::default())),
    MeshMaterial2d(materials.add(Color::WHITE)),
    Transform::default(),
)
```

# 四、核心组件和概念

## 4.1 Camera2d

`Camera2d` 启用二维渲染管线：

```rust
commands.spawn(Camera2d);
```

默认情况下，它使用正交投影：

- 没有近大远小的透视效果
- 世界中心通常对应窗口中心
- X 向右
- Y 向上
- Z 用于层级和深度

修改相机缩放：

```rust
fn zoom_camera(
    keyboard: Res<ButtonInput<KeyCode>>,
    mut camera: Single<&mut Projection, With<Camera2d>>,
) {
    let Projection::Orthographic(projection) = &mut *camera else {
        return;
    };

    if keyboard.pressed(KeyCode::KeyQ) {
        projection.scale *= 1.01;
    }

    if keyboard.pressed(KeyCode::KeyE) {
        projection.scale *= 0.99;
    }
}
```

投影的 `scale` 越大，看到的世界范围越大，画面中的物体越小。

---

## 4.2 Sprite

`Sprite` 用于绘制矩形图片或纯色矩形。

### 纯色 Sprite

```rust
Sprite::from_color(
    Color::srgb(0.2, 0.7, 1.0),
    Vec2::new(100.0, 60.0),
)
```

### 图片 Sprite

```rust
Sprite::from_image(asset_server.load("player.png"))
```

### 图集 Sprite

```rust
Sprite::from_atlas_image(
    image_handle,
    TextureAtlas {
        layout: layout_handle,
        index: 0,
    },
)
```

常用字段包括：

```rust
Sprite {
    image,
    texture_atlas,
    color,
    flip_x,
    flip_y,
    custom_size,
    rect,
    image_mode,
}
```

需要注意：

- `color` 会与贴图颜色相乘。
- `custom_size` 可以改变 Sprite 显示尺寸。
- `flip_x`、`flip_y` 用于水平或垂直翻转。
- `rect` 可以只显示贴图的一部分。
- `Anchor` 是独立组件，不是 `Sprite` 字段。

---

## 4.3 Transform

2D 和 3D 共用同一个 `Transform`：

```rust
Transform {
    translation: Vec3,
    rotation: Quat,
    scale: Vec3,
}
```

例如：

```rust
Transform::from_xyz(100.0, 50.0, 3.0)
    .with_rotation(Quat::from_rotation_z(0.5))
    .with_scale(Vec3::splat(2.0))
```

在 2D 中：

- `translation.x`：水平位置
- `translation.y`：垂直位置
- `translation.z`：视觉层级
- `rotation`：通常绕 Z 轴旋转
- `scale.x/y`：宽高缩放

建议统一定义层级：

```rust
const Z_BACKGROUND: f32 = -100.0;
const Z_WORLD: f32 = 0.0;
const Z_EFFECT: f32 = 50.0;
const Z_WORLD_TEXT: f32 = 100.0;
```

不要在业务代码中到处写不明含义的 `0.1`、`1.7`、`999.0`。

---

## 4.4 Text2d

世界空间文本示例：

```rust
commands.spawn((
    Text2d::new("Enemy"),
    TextFont {
        font_size: FontSize::Px(24.0),
        ..default()
    },
    TextColor(Color::WHITE),
    Transform::from_xyz(0.0, 50.0, 1.0),
));
```

还可以加入：

- `TextLayout`
- `TextBounds`
- `Anchor`
- `TextBackgroundColor`
- `Text2dShadow`

官方示例展示了 `Text2d` 的移动、旋转、缩放、换行和 Anchor。

---

## 4.5 Mesh2d

适合绘制：

- 圆形
- 矩形
- 多边形
- 自定义顶点
- 自定义 2D 材质

基本形式：

```rust
commands.spawn((
    Mesh2d(meshes.add(Circle::new(50.0))),
    MeshMaterial2d(materials.add(Color::srgb(0.8, 0.2, 0.3))),
    Transform::default(),
));
```

与 Sprite 相比：

- Sprite 更适合图片、角色、瓦片和矩形。
- Mesh2d 更适合程序化图形和自定义 Shader。
- 大量简单图片通常没有必要改用 Mesh2d。

---

## 4.6 TextureAtlas

图集由两部分组成：

```rust
Handle<Image>
Handle<TextureAtlasLayout>
```

`TextureAtlasLayout` 描述每一帧在图片中的区域：

```rust
let layout = TextureAtlasLayout::from_grid(
    UVec2::new(32, 32),
    4,
    2,
    None,
    None,
);
```

`TextureAtlas` 描述当前 Sprite 使用哪个布局和第几帧：

```rust
TextureAtlas {
    layout: layout_handle,
    index: 0,
}
```

动画的本质就是定时修改：

```rust
atlas.index
```

该示例不需要任何外部资源。

## Cargo.toml


## src/main.rs

```rust
use bevy::prelude::*;

const PLAYER_SPEED: f32 = 320.0;
const WINDOW_WIDTH: f32 = 960.0;
const WINDOW_HEIGHT: f32 = 540.0;

#[derive(Component)]
struct Player;

fn main() {
    App::new()
        .add_plugins(DefaultPlugins.set(WindowPlugin {
            primary_window: Some(Window {
                title: "Bevy 0.19 - 2D Basic".to_string(),
                resolution: (WINDOW_WIDTH as u32, WINDOW_HEIGHT as u32).into(),
                resizable: false,
                ..default()
            }),
            ..default()
        }))
        .add_systems(Startup, setup)
        .add_systems(Update, move_player)
        .run();
}

fn setup(mut commands: Commands) {
    // 2D 相机。
    commands.spawn(Camera2d);

    // 背景，Z 值最低。
    commands.spawn((
        Sprite::from_color(
            Color::srgb(0.06, 0.08, 0.13),
            Vec2::new(WINDOW_WIDTH - 20.0, WINDOW_HEIGHT - 20.0),
        ),
        Transform::from_xyz(0.0, 0.0, -10.0),
    ));

    // 中间的障碍物。
    commands.spawn((
        Sprite::from_color(
            Color::srgb(0.45, 0.18, 0.18),
            Vec2::new(220.0, 80.0),
        ),
        Transform::from_xyz(0.0, 80.0, 0.0),
    ));

    // 玩家。
    commands
        .spawn((
            Player,
            Sprite::from_color(
                Color::srgb(0.15, 0.65, 1.0),
                Vec2::new(72.0, 72.0),
            ),
            Transform::from_xyz(0.0, -120.0, 10.0),
        ))
        .with_children(|parent| {
            // 这是世界空间文字，会跟随玩家移动。
            parent.spawn((
                Text2d::new("PLAYER"),
                TextFont {
                    font_size: FontSize::Px(20.0),
                    ..default()
                },
                TextColor(Color::WHITE),
                Transform::from_xyz(0.0, 52.0, 1.0),
            ));
        });

    // 场景说明，同样是世界空间文字。
    commands.spawn((
        Text2d::new("Move with WASD or arrow keys"),
        TextFont {
            font_size: FontSize::Px(28.0),
            ..default()
        },
        TextColor(Color::WHITE),
        Transform::from_xyz(0.0, 220.0, 20.0),
    ));
}

fn move_player(
    keyboard: Res<ButtonInput<KeyCode>>,
    time: Res<Time>,
    mut player: Single<&mut Transform, With<Player>>,
) {
    let mut direction = Vec2::ZERO;

    if keyboard.pressed(KeyCode::KeyA)
        || keyboard.pressed(KeyCode::ArrowLeft)
    {
        direction.x -= 1.0;
    }

    if keyboard.pressed(KeyCode::KeyD)
        || keyboard.pressed(KeyCode::ArrowRight)
    {
        direction.x += 1.0;
    }

    if keyboard.pressed(KeyCode::KeyW)
        || keyboard.pressed(KeyCode::ArrowUp)
    {
        direction.y += 1.0;
    }

    if keyboard.pressed(KeyCode::KeyS)
        || keyboard.pressed(KeyCode::ArrowDown)
    {
        direction.y -= 1.0;
    }

    // 防止斜向移动比水平移动快。
    direction = direction.normalize_or_zero();

    player.translation.x +=
        direction.x * PLAYER_SPEED * time.delta_secs();
    player.translation.y +=
        direction.y * PLAYER_SPEED * time.delta_secs();

    // 将玩家限制在窗口范围内。
    let half_player_size = 36.0;
    let max_x = WINDOW_WIDTH * 0.5 - half_player_size - 10.0;
    let max_y = WINDOW_HEIGHT * 0.5 - half_player_size - 10.0;

    player.translation.x = player.translation.x.clamp(-max_x, max_x);
    player.translation.y = player.translation.y.clamp(-max_y, max_y);
}
```

```bash

```

# 六、完整可运行示例二：无外部图片的 TextureAtlas 动画

该示例会在内存中生成一张由四个彩色方块组成的图集，因此不需要准备 PNG 文件。

## 项目结构

```text

├── Cargo.toml
└── src/
    └── main.rs
```

## Cargo.toml

```toml
[package]

version = "0.1.0"
edition = "2021"

[dependencies]
bevy = "0.19.0"

[profile.dev]
opt-level = 1

[profile.dev.package."*"]
opt-level = 3
```

## src/main.rs

```rust
use bevy::{
    asset::RenderAssetUsages,
    prelude::*,
    render::render_resource::{
        Extent3d,
        TextureDimension,
        TextureFormat,
    },
};

const FRAME_SIZE: u32 = 32;
const FRAME_COUNT: u32 = 4;

#[derive(Component, Deref, DerefMut)]
struct AnimationTimer(Timer);

fn main() {
    App::new()
        .add_plugins(
            DefaultPlugins
                .set(ImagePlugin::default_nearest())
                .set(WindowPlugin {
                    primary_window: Some(Window {
                        title: "Bevy 0.19 - Texture Atlas".to_string(),
                        resolution: (800, 500).into(),
                        ..default()
                    }),
                    ..default()
                }),
        )
        .add_systems(Startup, setup)
        .add_systems(Update, animate_sprite)
        .run();
}

fn setup(
    mut commands: Commands,
    mut images: ResMut<Assets<Image>>,
    mut atlas_layouts: ResMut<Assets<TextureAtlasLayout>>,
) {
    commands.spawn(Camera2d);

    // 每一帧使用不同的 RGBA 颜色。
    let frame_colors: [[u8; 4]; FRAME_COUNT as usize] = [
        [255, 80, 80, 255],
        [255, 210, 70, 255],
        [80, 220, 120, 255],
        [80, 150, 255, 255],
    ];

    let image_width = FRAME_SIZE * FRAME_COUNT;
    let image_height = FRAME_SIZE;

    let mut pixel_data =
        Vec::with_capacity((image_width * image_height * 4) as usize);

    // 生成一张横向排列四帧的图片。
    for _y in 0..image_height {
        for x in 0..image_width {
            let frame_index = (x / FRAME_SIZE) as usize;
            pixel_data.extend_from_slice(&frame_colors[frame_index]);
        }
    }

    let image = Image::new(
        Extent3d {
            width: image_width,
            height: image_height,
            depth_or_array_layers: 1,
        },
        TextureDimension::D2,
        pixel_data,
        TextureFormat::Rgba8UnormSrgb,
        RenderAssetUsages::default(),
    );

    let image_handle = images.add(image);

    // 描述图集中每一帧的位置。
    let layout_handle = atlas_layouts.add(
        TextureAtlasLayout::from_grid(
            UVec2::splat(FRAME_SIZE),
            FRAME_COUNT,
            1,
            None,
            None,
        ),
    );

    commands.spawn((
        Sprite::from_atlas_image(
            image_handle,
            TextureAtlas {
                layout: layout_handle,
                index: 0,
            },
        ),
        Transform::from_scale(Vec3::splat(5.0)),
        AnimationTimer(Timer::from_seconds(
            0.18,
            TimerMode::Repeating,
        )),
    ));

    commands.spawn((
        Text2d::new("Programmatically generated texture atlas"),
        TextFont {
            font_size: FontSize::Px(26.0),
            ..default()
        },
        TextColor(Color::WHITE),
        Transform::from_xyz(0.0, -140.0, 1.0),
    ));
}

fn animate_sprite(
    time: Res<Time>,
    mut sprites: Query<(&mut AnimationTimer, &mut Sprite)>,
) {
    for (mut timer, mut sprite) in &mut sprites {
        timer.tick(time.delta());

        if !timer.just_finished() {
            continue;
        }

        let Some(atlas) = sprite.texture_atlas.as_mut() else {
            continue;
        };

        atlas.index =
            (atlas.index + 1) % FRAME_COUNT as usize;
    }
}
```

## src/main.rs

```rust
use bevy::prelude::*;

const NORMAL_BUTTON: Color =
    Color::srgb(0.15, 0.15, 0.18);

const HOVERED_BUTTON: Color =
    Color::srgb(0.25, 0.25, 0.32);

const PRESSED_BUTTON: Color =
    Color::srgb(0.20, 0.60, 0.30);

#[derive(Component)]
struct Player;

#[derive(Component)]
struct ChangeColorButton;

#[derive(Resource, Default)]
struct PlayerColorState {
    alternate: bool,
}

fn main() {
    App::new()
        .add_plugins(DefaultPlugins.set(WindowPlugin {
            primary_window: Some(Window {
                title: "Bevy 0.19 - 2D and UI".to_string(),
                resolution: (900, 560).into(),
                ..default()
            }),
            ..default()
        }))
        .init_resource::<PlayerColorState>()
        .add_systems(Startup, setup)
        .add_systems(Update, handle_button)
        .run();
}

fn setup(mut commands: Commands) {
    commands.spawn(Camera2d);

    // 这是 2D 世界对象。
    commands.spawn((
        Player,
        Sprite::from_color(
            Color::srgb(0.15, 0.65, 1.0),
            Vec2::splat(150.0),
        ),
        Transform::from_xyz(0.0, 0.0, 0.0),
    ));

    // 这是 UI 文本，使用窗口左上角进行定位。
    commands.spawn((
        Text::new("This text belongs to the UI system"),
        TextFont {
            font_size: FontSize::Px(25.0),
            ..default()
        },
        TextColor(Color::WHITE),
        Node {
            position_type: PositionType::Absolute,
            top: px(18),
            left: px(18),
            ..default()
        },
    ));

    // 这是 UI 按钮。
    commands.spawn((
        ChangeColorButton,
        Button,
        Node {
            position_type: PositionType::Absolute,
            left: px(20),
            bottom: px(20),
            width: px(250),
            height: px(64),
            border: UiRect::all(px(3)),
            justify_content: JustifyContent::Center,
            align_items: AlignItems::Center,
            ..default()
        },
        BackgroundColor(NORMAL_BUTTON),
        BorderColor::all(Color::WHITE),
        children![(
            Text::new("Change sprite color"),
            TextFont {
                font_size: FontSize::Px(22.0),
                ..default()
            },
            TextColor(Color::WHITE),
        )],
    ));
}

fn handle_button(
    mut state: ResMut<PlayerColorState>,
    mut buttons: Query<
        (&Interaction, &mut BackgroundColor),
        (
            Changed<Interaction>,
            With<ChangeColorButton>,
        ),
    >,
    mut player: Single<&mut Sprite, With<Player>>,
) {
    for (interaction, mut background) in &mut buttons {
        match *interaction {
            Interaction::Pressed => {
                state.alternate = !state.alternate;

                player.color = if state.alternate {
                    Color::srgb(1.0, 0.35, 0.25)
                } else {
                    Color::srgb(0.15, 0.65, 1.0)
                };

                *background =
                    BackgroundColor(PRESSED_BUTTON);
            }

            Interaction::Hovered => {
                *background =
                    BackgroundColor(HOVERED_BUTTON);
            }

            Interaction::None => {
                *background =
                    BackgroundColor(NORMAL_BUTTON);
            }
        }
    }
}
```

## 为什么重力设置为 900

Avian 默认使用类似“米”的物理尺度，而很多 2D 游戏使用“像素”作为世界单位。

在像素坐标中：

```rust
9.81
```

每秒平方像素几乎看不出明显下落，因此示例使用：

```rust
Gravity(Vec2::NEG_Y * 900.0)
```

正式项目也可以定义转换比例，例如：

```text
100 像素 = 1 米
```

这样物理参数会更容易管理。

# 九、鼠标点击 2D 世界对象

2D 世界中的鼠标坐标首先是窗口坐标，不能直接与 `Transform.translation` 比较。

需要使用：

```rust
camera.viewport_to_world_2d(
    camera_global_transform,
    cursor_position,
)
```

基本逻辑如下：

```rust
fn cursor_world_position(
    camera: Single<(&Camera, &GlobalTransform)>,
    window: Single<&Window>,
) -> Option<Vec2> {
    let cursor_position = window.cursor_position()?;
    let (camera, camera_transform) = *camera;

    camera
        .viewport_to_world_2d(
            camera_transform,
            cursor_position,
        )
        .ok()
}
```

这会自动考虑：

- 相机位置
- 相机缩放
- Viewport
- 窗口尺寸
- 正交投影

官方示例同样使用 `Camera::viewport_to_world_2d` 完成窗口坐标到世界坐标的转换。

简单矩形点击检测：

```rust
fn point_inside_sprite(
    point: Vec2,
    transform: &GlobalTransform,
    size: Vec2,
) -> bool {
    let center = transform.translation().truncate();
    let half_size = size * 0.5;

    point.x >= center.x - half_size.x
        && point.x <= center.x + half_size.x
        && point.y >= center.y - half_size.y
        && point.y <= center.y + half_size.y
}
```

复杂项目可以使用：

- Bevy Picking
- `sprite_picking` 功能
- Avian Spatial Query
- Rapier QueryPipeline

# 十、2D 与 UI 的区别

| 对比项     | 2D 世界系统                               | UI 系统                                |
| ---------- | ----------------------------------------- | -------------------------------------- |
| 主要组件   | `Sprite`、`Mesh2d`、`Text2d`、`Transform` | `Node`、`Button`、`Text`、`ImageNode`  |
| 定位方式   | 世界坐标                                  | 布局坐标                               |
| 常用单位   | 世界单位，项目中常按像素理解              | `px()`、`percent()`、`vw()`、`vh()` 等 |
| 布局能力   | 没有自动 Flex/Grid 布局                   | 支持 Flexbox 和 CSS Grid               |
| 相机影响   | 会随 `Camera2d` 移动和缩放                | 通常作为屏幕界面显示                   |
| 旋转       | 使用 `Transform`                          | UI 主要以布局和 UI Transform 为主      |
| 层级       | 常通过 `Transform.translation.z`          | 主要由 UI 树、绘制顺序和 UI 层级控制   |
| 交互       | 自行做坐标转换或使用 Picking              | 内置 `Interaction`                     |
| 文本       | `Text2d`                                  | `Text`                                 |
| 适用对象   | 玩家、敌人、地图、特效、世界标签          | 菜单、按钮、HUD、背包、弹窗            |
| 相机缩放   | 对对象产生明显影响                        | 一般不随世界相机缩放                   |
| 响应式布局 | 需要自己编写                              | 支持百分比、Flex 和 Grid               |

## 判断原则

### 应该使用 2D 的情况

对象属于“游戏世界”：

- 玩家
- 子弹
- 地图
- 敌人
- 伤害数字
- 场景中的名字
- 世界里的提示标记
- 跟随相机移动的对象

### 应该使用 UI 的情况

对象属于“屏幕界面”：

- 生命值条
- 背包
- 菜单
- 设置面板
- 暂停界面
- 固定在屏幕角落的小地图
- 按钮
- 文本输入框

## 常见混合方式

```text
Camera2d
├── 世界背景 Sprite
├── 地图 TilemapChunk
├── 玩家 Sprite
├── 敌人 Sprite
└── 世界空间 Text2d

UI
├── 血量
├── 技能栏
├── 小地图
├── 暂停按钮
└── 菜单
```

UI 与 2D 可以共用同一个 `Camera2d`，但它们仍属于不同的布局和渲染语义。官方 UI 示例也会生成 `Camera2d`，再创建由 `Node`、`Button` 和 `Text` 组成的 UI 树。

# 十一、2D 与 3D 的区别

| 对比项   | 2D                                   | 3D                             |
| -------- | ------------------------------------ | ------------------------------ |
| 相机     | `Camera2d`                           | `Camera3d`                     |
| 默认投影 | 正交投影                             | 透视投影                       |
| 基本对象 | Sprite 或平面 Mesh                   | 三维 Mesh                      |
| 常用组件 | `Sprite`、`Mesh2d`、`MeshMaterial2d` | `Mesh3d`、`MeshMaterial3d`     |
| 基础材质 | `ColorMaterial`                      | `StandardMaterial`             |
| 光照     | Sprite 通常不依赖场景灯光            | 通常使用方向光、点光源等       |
| 坐标     | X/Y 是主要平面，Z 常用于层级         | X/Y/Z 都表示空间位置           |
| 旋转     | 通常只绕 Z 轴                        | 可以绕任意轴旋转               |
| 深度关系 | 常显式规划 Z 层级                    | 依赖空间深度和深度缓冲         |
| 物理     | 圆、矩形、多边形等 2D 碰撞体         | 球、立方体、胶囊、三角网格等   |
| 资源     | 精灵图、图集、瓦片                   | 模型、骨骼、PBR 贴图、环境贴图 |
| 常见用途 | 平面游戏、编辑器、可视化             | 3D 游戏、仿真、建筑和空间场景  |

Bevy 0.19 官方 3D 示例直接使用：

```rust
Mesh3d
MeshMaterial3d<StandardMaterial>
PointLight
Camera3d
```

而不是旧版 `PbrBundle` 和 `Camera3dBundle`。

## 11.1 2D 是否一定比 3D 性能更好

不一定。

2D 通常具有以下优势：

- 每个 Sprite 几何结构简单
- 通常不计算 PBR 光照
- 不需要复杂三维模型
- 碰撞形状通常更简单
- 动画资源结构通常更简单

但 2D 也可能产生严重性能问题：

- 数十万个 ECS 实体
- 大面积透明 Sprite 导致 Overdraw
- 大量不同贴图造成材质或批次切换
- 每帧修改大量 UI 节点
- 每个瓦片都单独建立实体
- 使用大量全屏透明特效
- 频繁创建和销毁粒子实体

因此不能简单认为“2D 天然高性能”。

---

## 11.2 什么时候应该从 2D 转向 3D

出现以下需求时，应考虑 3D：

- 明显的透视关系
- 需要三维摄像机环绕
- 使用 3D 模型
- 需要实时灯光和阴影
- 角色要在真实三维空间中移动
- 使用骨骼模型和 PBR 材质
- 需要复杂三维物理

仍适合使用 2D 的情况：

- 游戏逻辑本质是平面的
- 美术全部为 Sprite
- 深度只用于前后层级
- 不需要透视和三维光照
- 关卡编辑基于二维网格

---

## 11.3 2.5D

2.5D 常见形式包括：

### 3D 世界加 UI

```text
Camera3d：渲染 3D 世界
UI：渲染 HUD、菜单和准星
```

### 3D 世界加 2D Sprite

例如：

- 3D 场景中的名字标签
- Billboard 血条
- 2D 特效
- 纸片角色

### 2D 玩法加 3D 渲染

例如：

- 游戏逻辑只在 X/Y 平面移动
- 场景和角色使用 3D 模型
- 相机保持固定角度

这种情况下通常仍使用 `Camera3d`，但限制角色在一个平面内移动。

# 十三、Update 与 FixedUpdate

## Update

适合：

- UI
- 动画显示
- 相机跟随
- 根据输入修改意图
- 不要求固定频率的逻辑

```rust
app.add_systems(Update, animate_sprite);
```

## FixedUpdate

适合：

- 物理
- 固定时间步移动
- 可重放游戏逻辑
- 对时间稳定性要求高的模拟

```rust
app.add_systems(FixedUpdate, simulate_movement);
```

推荐将输入和运动意图分离：

```text
Update
└── 读取键盘，写入 MovementInput

FixedUpdate
└── 根据 MovementInput 更新速度和物理状态
```

这样可以避免输入采样频率与物理频率直接耦合。

# 十四、性能建议

## 14.1 大型地图不要每格一个 Sprite

小地图可以这样做，但大型地图应优先考虑：

- `TilemapChunk`
- Chunk 分区
- 可见区域加载
- 自定义实例化渲染
- 专门的 Tilemap 插件

Bevy 0.19 的 `TilemapChunk` 可以将一个 Chunk 作为单次绘制调用提交。

## 14.2 控制透明 Overdraw

大面积透明图片会造成 GPU 重复绘制。

尤其要避免：

- 多层全屏半透明雾
- 大量互相覆盖的透明粒子
- 图片实际内容很小，但透明边界很大

## 14.3 使用图集

将角色帧、小图标和瓦片合并到图集中，可以：

- 降低贴图切换
- 方便动画管理
- 改善资源加载组织

但不要将所有游戏图片强行放进一张超大图集，否则可能造成：

- 加载粒度过大
- 无法按需卸载
- 纹理尺寸超限
- 修改任何内容都要重建整张图集

## 14.4 避免无意义地频繁生成实体

粒子或子弹数量较多时，可以考虑：

- 对象池
- 批量生成
- 限制最大数量
- 超出视野后及时清理
- 使用 GPU 粒子插件

## 14.5 不要每帧修改所有组件

Bevy 的变更检测依赖组件是否发生修改。

系统查询可以使用：

```rust
Changed<Transform>
Added<Enemy>
With<Player>
Without<Disabled>
```

减少不必要的全世界扫描。

| 需求                   | 推荐方案                              |
| ---------------------- | ------------------------------------- |
| 显示一张图片           | `Sprite::from_image`                  |

# 十六、结论

Bevy 0.19 的 2D 开发应围绕直接组合组件来组织：

```text
Camera2d
Sprite / Mesh2d / Text2d
Transform
自定义 ECS 组件
系统
资源
Plugin
```

最重要的边界是：

- **属于游戏世界的对象使用 2D 系统。**
- **属于屏幕界面的对象使用 UI 系统。**
- **需要真实空间、透视、灯光和三维模型时使用 3D 系统。**

对于多数 2D 游戏，推荐组合是：

```text
Sprite             角色、敌人和物品
TextureAtlas       帧动画
TilemapChunk       大型地图
Text2d             世界标签
UI                 HUD 和菜单
Avian 2D           物理与碰撞
Plugin             模块化组织
```

这套结构与 Bevy 0.19 当前官方示例采用的直接组件式 API 保持一致。
